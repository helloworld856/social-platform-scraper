# "检查更新"功能实施计划

## 概要

为 PyQt5 桌面工作台增加启动时检查 GitHub Releases 更新功能。
有新版时在右上角显示可点击提示，支持热更新（打开浏览器下载）。

### 步骤 2：新建 `src/core/updater.py` —— 封装更新检查核心逻辑

**文件**: `src/core/updater.py`（新建）

实现函数：
```python
def check_for_updates(current_version: str, repo_owner: str, repo_name: str) -> tuple[bool, str | None, str | None]:
    """
    检查 GitHub 是否有新版本。

    Returns:
        (has_update, latest_version, release_url)
        - has_update: True 表示存在新版本
        - latest_version: 远程最新版本号（不含 v 前缀），如 "2.1.0"；无 release 时为 None
        - release_url: 新版本的 html_url；无 release 时为 None
    """
```

逻辑细节：
1. 构造请求 `GET https://api.github.com/repos/{owner}/{repo}/releases/latest`
2. 设置 `User-Agent: social-platform-scraper`（GitHub API 要求）
3. 超时 5 秒 (`timeout=5`)
4. 响应处理：
   - `200`: 从 `tag_name` 字段提取版本号，去掉前缀 `v`
   - `404`: 该仓库没有 release → 不更新
   - 其他状态码 / 网络异常: 向外抛出，由调用方统一处理
5. 版本比较：将 `major.minor.patch` 拆分为三元组 `(int, int, int)` 逐位比较
   - 兼容非标准格式：解析失败时退回字符串 `>` 比较
6. 返回 `(has_update, latest_version, release_url)`

#### 3c. 实现 `_check_for_updates` 方法
```python
def _check_for_updates(self) -> None:
    """后台线程检查版本更新。"""
    from src.version import __version__
    from src.core.updater import check_for_updates
    import threading

    def _worker():
        try:
            has_update, latest, url = check_for_updates(
                __version__, "helloworld856", "social-platform-scraper"
            )
            if has_update and latest and url:
                self._show_update_banner(latest, url)
        except Exception as e:
            logger.warning("检查更新失败: %s", e)
            # 在主线程弹出错误提示
            QTimer.singleShot(0, lambda: self._show_update_error(str(e)))

    t = threading.Thread(target=_worker, daemon=True)
    t.start()
```

#### 3d. 实现 UI 回调
```python
def _show_update_banner(self, latest_version: str, url: str) -> None:
    """在主窗口右上角显示更新提示。"""
    text = f'发现新版本 v{latest_version}，当前版本为 {__version__}，<a href="{url}">点击下载</a>'
    self.update_label.setText(text)
    self.update_label.setVisible(True)

def _on_update_clicked(self, url: str) -> None:
    """用系统默认浏览器打开下载页面。"""
    import webbrowser
    webbrowser.open(url)

def _show_update_error(self, detail: str) -> None:
    """弹出检查更新失败对话框。"""
    QMessageBox.warning(self, "检查更新失败", "检查更新失败，请稍后重试。")
```

## 风险与注意事项

1. **GitHub API 限流**：未认证请求每小时 60 次。首次检查 + 手动触发控制在合理范围。
2. **无 release 场景**：仓库当前无任何 release 时返回 404，代码已处理为「无更新」。
3. **线程安全**：网络请求在 daemon 线程执行，UI 更新通过 `QTimer.singleShot(0, ...)` 回主线程。
4. **`requests` 依赖**：已是 google-api-python-client 的传递依赖，无需新增 `requirements.txt` 条目。
